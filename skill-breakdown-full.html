IMG-C1 SVG 矢量插画生成
P0✅ 2天可交付
▾
输入主题描述，直接生成可编辑的 SVG 矢量插画代码，适用于 UI 配图、空状态页、品牌图形。
输入
  • 主题描述（如：「空购物车」「上传成功」「团队协作」）
  • 风格偏好（可选：Flat / Line / Isometric）
  • 色板（可选，如品牌主色 #1677FF）
  • 尺寸（可选，默认 400×300）
输出
  • 完整 SVG 代码（可直接嵌入 HTML/React）
  • 可在 Figma/Illustrator 中进一步编辑
  • 文件大小通常 <10KB
意图路由设计
用户输入 → 意图判断层
├── 需要写实感 → IMG-D1（文生图）
├── 需要可编辑矢量 → IMG-C1（本 Skill）
└── 有现成图改风格 → IMG-D3（图生图）
质量标准（竞品对标：GPT-4o SVG 输出）
维度标准
语义准确性图形内容与主题描述匹配，一眼可识别
代码合法性标准 SVG 语法，浏览器直接渲染无报错
视觉质量构图平衡、颜色协调，不是纯线框
可编辑性SVG 元素有语义化命名，便于后续调整
  • 不做照片级写实（那是 Diffusion 的工作）
  • 复杂 3D 透视插画效果有限（Flat/Line 风格为主）
  • 不保证每次生成一致（非精确复现场景）
★★★★★可行性：高 — GPT-4o/Claude 3.5 SVG 能力已验证

VID-F1 HTML+CSS 微动效生成（L1）
P0✅ 2天可交付
▾
输入 UI 元素描述，直接生成可内嵌的 CSS 动效代码，适用于按钮交互、Loading、背景动效等 L1 场景。
输入
  • 动效元素描述（如：「按钮点击涟漪效果」「骨架屏加载」「渐入渐出页面元素」）
  • 持续时间（可选，默认 300-500ms）
  • 触发方式（hover / click / 页面加载 / 循环）
  • 目标框架（可选，纯CSS / Tailwind / React inline）
输出
  • 可运行的 HTML+CSS 代码片段
  • 可选：Codepen 可运行 Demo 格式
  • 动效参数说明（duration / easing / delay）
L1 动效类型清单（本 Skill 覆盖范围）
类型示例难度
悬停反馈按钮 scale/shadow 变化⭐
加载状态Spinner、Skeleton、Shimmer⭐⭐
页面入场元素淡入、从下滑入⭐
背景动效渐变流动背景、粒子漂浮⭐⭐
数字滚动Counter 动效⭐⭐
质量标准（竞品对标：Framer Motion 官方示例）
维度标准
代码可用直接粘贴可运行，无语法错误
动效自然度缓动函数合理（非线性 ease），不生硬
性能优先使用 transform/opacity（GPU 加速），不触发 layout
参数可调关键参数（时长/延迟/颜色）抽成变量，便于修改
  • 不支持需要 JS 逻辑联动的复杂序列动效（→ F2/G1）
  • 不做视频导出（纯 CSS，非 Lottie/视频格式）
★★★★★可行性：高 — LLM 生成 CSS 动效是最成熟的能力之一

VID-G1 Lottie JSON 动效生成 🔬
PoC❌ 需 PoC
▾
输入动效描述，直接生成 Lottie JSON 文件，可在 iOS/Android/Web 直接渲染，适用于 App Loading、品牌动效、空状态动画等 L2 场景。
背景说明：Lottie 是 App 端动效的标准格式（JSON，约 50KB）。现有 Sora/Kling 等模型生成的是视频（MP4），无法直接在 App 内运行。OmniLottie（2026.03 发布，CVPR 2026）是首个能直接生成 Lottie JSON 的 AI 模型，已开源（arxiv 2603.02138）。
技术路线
OmniLottie 模型（CVPR 2026 开源）/ LottieFiles AI Prompt to Vector，需评估接入可行性和工程成本
★★★可行性：中等 — 技术前沿，需要工程评审，建议先做 PoC
