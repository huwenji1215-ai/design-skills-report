Known Failures 沉淀
本次评测发现的问题模式，写入飞轮供下批 Skill 准入规范参考
[AQE-E × DIR-01] 可访问性验证缺失
失败类型: AQE-E（可访问性）· UI生成方向
触发条件: 模型代码过度压缩（<100行），以代码简洁为由省略 WCAG 对比度验证标注
症状: AQE-E 维度 ≤ 6.0，无 WCAG 注释，状态标识仅靠颜色区分
根因: Skill description 中虽有 WCAG AA 要求，但代码量有限时模型优先删减注释
修复指令: 在 description 中增加「无论代码长短，必须在 CSS 变量中注释各文字色的对比度数值」
防重现规则: 新增准入 Checklist 条目「P0：主要文字色变量必须有对比度注释（如 /* 对比度 7.8:1 */）」
[AQE-B × DIR-01] 情感色温判断失误
失败类型: AQE-B（色彩美学）· UI生成方向
触发条件: 用户指定「暖色调」但未精确约束色温范围
症状: 模型选用 #D8432A（深砖红）作为主色，符合「暖色」但偏向警示/危险语义，与「疗愈/情绪记录」调性冲突
根因: 「暖色调」覆盖范围宽（橙、红、黄），需要更精细的情感约束
修复指令: Skill description 中建议增加「当产品类型为 consumer_app（情感/疗愈类），暖色调优先推荐橙系（#FF8A65-#FFB74D），避免使用红系（>=#E53935）」
防重现规则: 新增准入 Checklist 条目「P1：consumer_app 类产品默认主色 Hue 在 10°-50°（橙黄系），需在 description 中显式约束色相范围」
[结构规范] SKILL.md 超长问题
失败类型: 结构合规 · SKILL.md 行数超标
触发条件: 内联大量代码模板（HTML结构、CSS片段、BUG索引表）导致单文件超 1700 行
症状: SKILL.md 1789 行，超出 500 行规范 3.6x，注意力衰减风险高
根因: Phase 3 代码模板（快速模板片段 800+ 行）、QA BUG 索引表等直接嵌在 SKILL.md 中
修复指令: 将「快速模板片段」移至 references/component-library.md，将「BUG 索引表」移至 references/ux-patterns.md，SKILL.md 保留核心工作流和决策规则
防重现规则: 新增准入 Checklist 条目「P0：SKILL.md 不得超过 600 行，代码模板统一放 references/ 目录，SKILL.md 以 read 指令引用」
