Section 07
Known Failures · 失败案例沉淀
本次评测发现的可复现失败模式，写入 Known Failures 库，注入下批 Skill 准入规范
KF-001 · Toast提示在所有模型中均缺失
AQE-C · UI生成 失败类型：版式规范 + 交互完整性
触发条件：调用任意模型生成导出功能时
症状：导出 PNG/JPG 时无 loading 提示和完成反馈，用户体验断层
根因：SKILL.md 规范4中 toast 要求优先级标记不明显（藏在导出实现细节中，未单独列为强制项）
修复指令：在 SKILL.md 「三大强制输出规范」中新增「规范5：Toast反馈」，提升为与编辑面板同级的强制项
防重现规则：下批 Skill 准入时，「产物含导出功能 → 必须包含 Toast 反馈」写入 UI生成方向 Checklist
KF-002 · Kimi K2 绝对定位导致布局不稳定
AQE-C · UI生成 失败类型：版式规范 · 间距一致性
触发条件：模型代码量小（约6KB）、精简实现时
症状：按钮使用 position:absolute; right:300px 硬编码，在不同屏宽下错位
根因：SKILL.md 未明确禁止绝对定位布局，缺少「禁止使用硬编码像素定位」的约束指令
修复指令：在代码规范章节增加「禁止使用 position:absolute 实现功能性布局，必须使用 flex/grid」
防重现规则：合规检查脚本中增加对 absolute 定位+像素值组合的检测
KF-003 · CSS选择器顺序错误导致主题切换失效
AQE-D · UI生成 失败类型：品牌一致性 · 跨次风格稳定
触发条件：模型生成浅色主题 CSS 时
症状：Kimi K2 使用 .light body 选择器（CSS优先级低于 body.light），亮色主题切换可能无效
根因：SKILL.md 代码模板中只有 :root 和 .theme-light 类名方案，未明确说明主题切换的正确 CSS 选择器规范
修复指令：在代码模板注释中增加「⚠️ 主题类必须加在 html 或 body 标签上：body.theme-light {} 或 [data-theme="light"] {}」
防重现规则：新增测试用例：切换主题后检查背景色是否实际变化
KF-004 · Claude Opus 4.7 标题字号层级不足
AQE-A · UI生成 失败类型：视觉层级与信息架构
触发条件：默认图表标题样式生成时
症状：图表主标题 20px，副标题 13px，层级比例不足（建议 28/16），商务汇报场景中重要信息不够突出
根因：SKILL.md 代码模板中 .chart-title { font-size:18px } 字号偏小，未与副标题形成足够对比
修复指令：将代码模板中 .chart-title 字号从 18px 改为 24px，.chart-subtitle 从默认调为 13px，明确 1:0.55 的比例约束
防重现规则：准入 Checklist 新增「主副标题字号比例 ≥ 1.5:1」检查项
