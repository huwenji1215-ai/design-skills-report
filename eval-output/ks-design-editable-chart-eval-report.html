KF-001 · AQE-C · DIR-01 · UI生成
Toast提示在所有模型中均缺失（共性失败）
失败类型
AQE-C（版式规范·交互完整性）× DIR-01 UI生成
触发条件
生成含导出功能（PNG/JPG）的 HTML 图表时
症状
Claude/Kimi/Gemini 三个模型均未在导出时实现 Toast 反馈（loading 提示 + 完成通知），用户点击导出后无任何状态反馈
根因
SKILL.md 规范4中 toast 要求描述层级低，藏在导出实现细节中，未单独列为强制项（无独立编号、无P0标记），导致3个模型一致遗漏
修复指令
在 SKILL.md 「三大强制输出规范」中新增「规范5：Toast反馈」，明确提升为强制项；在代码模板中增加 showToast() 函数示例
防重现规则
准入 Checklist 新增：「产物含导出功能 → 必须包含Toast反馈（loading + 完成）」作为P0检查项
KF-002 · AQE-C · DIR-01 · Kimi K2
绝对定位硬编码导致布局在不同屏宽下错位
失败类型
AQE-C（版式规范·间距一致性）× DIR-01 UI生成
触发条件
模型代码量小（约6KB）、追求精简实现时，倾向用绝对定位代替flex布局
症状
Kimi K2 产物中按钮使用 position:absolute; right:300px 和 right:360px 硬编码，在不同浏览器宽度下按钮错位覆盖图表区域
根因
SKILL.md 未明确禁止绝对定位布局，未声明「布局必须使用flex/grid」的约束指令
修复指令
代码规范章节增加：「禁止使用 position:absolute 实现功能性布局，必须使用 flex 或 grid；px 硬编码偏移值不得用于响应式布局定位」
防重现规则
产物合规检查中增加：检测 absolute 定位+像素值组合；Gate 2 Checklist 新增「布局使用flex/grid（非absolute定位）」检查项
KF-003 · AQE-D · DIR-01 · Kimi K2
CSS主题选择器顺序错误导致浅色主题切换失效
失败类型
AQE-D（品牌一致性·跨次风格稳定）× DIR-01 UI生成
触发条件
模型生成双主题CSS时，使用 .light body {} 选择器而非 body.light {} 格式
症状
Kimi K2 产物的 .light body 选择器CSS优先级低于默认 body 规则，点击切换至亮色主题后背景色可能无法生效
根因
SKILL.md 代码模板中只展示了 :root 变量覆盖方案，未明确说明主题切换的正确CSS选择器写法规范
修复指令
代码模板注释中增加：「⚠️ 主题类必须加在 html 或 body 标签上：body.theme-light {} 或 [data-theme="light"] {}，禁止使用 .class body {} 写法」
防重现规则
新增测试用例：切换主题后取 body 的 backgroundColor 计算值，验证是否实际变化
KF-004 · AQE-A · DIR-01 · Claude Opus 4.7
图表主标题字号偏小，与副标题对比度不足
失败类型
AQE-A（视觉层级·信息架构）× DIR-01 UI生成
触发条件
Claude 使用默认 chart-title 样式时，字号偏保守
症状
Claude 产物主标题 20px，副标题 13px，比例1:0.65，层级感不足；汇报场景中核心信息不够突出，与 Gemini（主标题更大）相比差距明显
根因
SKILL.md 代码模板中 .chart-title { font-size: 18px } 字号偏小，未明确「主副标题字号比例 ≥ 1.5:1」约束
修复指令
将代码模板中 .chart-title 字号从 18px 改为 24px，.chart-subtitle 保持13px，形成1:0.54比例；准入Checklist新增「主副标题字号比例 ≥ 1.5:1」
防重现规则
新增自动检查：提取产物中 .chart-title 和 .chart-subtitle 的 font-size，验证比值 ≥ 1.5
