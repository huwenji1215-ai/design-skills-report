Mock Case 设计说明

📋 Case 基本信息
Case 名称内容审核后台 · 待审核内容列表页
业务场景内容平台内容安全审核团队使用的审核工作台，审核员需要查看待审核内容列表、快速判断风险等级并执行通过/下架操作
页面类型ListPage（后台管理列表页）· 复合型，含统计卡 + 表格 + 筛选 + 操作弹层
选择理由KS-AI-UI 核心模板之一（ListPage），业务复杂度中等，包含：侧边导航、统计面板、多条件筛选、数据表格、行操作、弹层/Toast 等高频 UI 元素，适合全面考察 skill 的组件覆盖能力与设计规范落地能力
目标用户内容安全审核员（内部工具用户），高频操作、低容错、需要快速扫描信息
模型数量3 个（Model A · Model B · Model C），每个模型使用同一 prompt，输出风格自然分叉
🖥️ 页面核心需求（输入给三个模型的统一需求描述）
以下需求作为 Mock Case 的统一输入，三个模型在相同需求下生成差异化的 UI 产物：
业务背景：内容平台审核工作台，内容安全团队使用。 需要包含的功能区块： 1. 页面整体布局：带侧边导航 + 顶部导航的后台 Layout 2. 统计卡区域：至少展示 待审核总量、高风险数量、今日已处理、平均处理时长 3. 筛选与操作栏：内容类型（视频/图片/文本/直播）+ 风险等级（高/中/低）筛选；批量处理、导出、刷新按钮 4. 数据表格： - 列：内容缩略图+标题+作者、内容类型标签、风险等级标识、AI 置信度进度条、违规标签 Chips、提交时间、状态、操作 - 每行操作：详情（打开弹层）、审核通过（绿色） - 多选 Checkbox 5. 详情弹层/抽屉：展示内容详情 + AI 审核意见 + 处理按钮（通过/下架/跳过） 6. 交互反馈：操作后展示成功提示（Toast/Snackbar） 7. 分页组件 数据规模：模拟 10 条数据（涵盖高/中/低风险，视频/图片/文本/直播四种类型） 技术要求：单 HTML 文件，无外部依赖，可在浏览器直接打开，数据用 JS 对象 mock

检查项要求实际情况评级优先级
DIR-01-A 组件库声明 SKILL.md 中明确声明使用的组件库（Ant Design/Material-UI/shadcn…）及版本 SKILL.md 提到 Ant Design Pro，但未在 description 或 dependencies.md 中明确声明版本号 ❌ FAIL P0
DIR-01-B 响应式策略 声明是否支持响应式，若支持需说明断点策略 SKILL.md 未提及响应式支持；模板文件中未见 @media 查询或响应式布局说明 ⚠️ PARTIAL P1
