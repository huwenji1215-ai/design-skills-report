Known Failures · 失败案例沉淀

本次评测发现的失败模式，按标准格式写入 Known Failures 库。

KF-001 · AQE-E · DIR-PPT · 暗色主题通病
次级文字色（--text-muted）在暗色背景下对比度系统性不足
失败类型
AQE-E1（文字对比度）× 三模型均出现
触发条件
幻灯片使用暗色 Palette（A/F/B），卡片内使用 --text-muted 级别描述文字
症状
Model-A node-desc ≈3.8:1；Model-B uc-desc ≈3.2:1；Model-C ci-label ≈3.9:1；均低于 WCAG AA 4.5:1
根因
各 Palette 的 --text-muted 色值设计时未做对比度计算，仅凭视觉感受定值；Stage 4 QA 未做对比度数值验证
修复指令
各暗色 Palette --text-muted 值需满足 4.5:1 基线，建议：Palette-A #4a6fa5→#6b90cc；Palette-F #6b5e42→#8a7a55；Palette-B #7a5535→#9a7a55
防重现
Stage 4 QA 中新增：检查所有次级文字色与其实际背景的对比度是否 ≥ 4.5:1
KF-002 · Rule 40 · DIR-PPT · Model-B
布局多样性不足：连续两页均为「多列卡片组」形态
失败类型
Rule 40 EACH_SLIDE_UNIQUE 违规 × Palette-F 模型
触发条件
封面右侧选用「统计卡片竖列」，下一张选用「场景矩阵卡片网格」，两页主体布局形态均为卡片组
症状
两张幻灯片在视觉上感知相似度高，缺乏翻页节奏感，违反 Rule 40 设计意图
根因
slides.yml 生成阶段 layout_diversity_rule 检查粒度不够细——仅约束 layout 类型名称，未约束「布局视觉形态」相似度
修复指令
在 Stage 1 slides.yml 生成规则中补充：连续两页的主体布局视觉形态不得均为「卡片网格」类型，须穿插时间轴/单栏叙述/图文对比等形态
防重现
Stage 4 QA 新增：浏览前 N 页，是否有超过 2 页连续使用相似卡片网格布局？
KF-003 · Gate 0 · DIR-PPT · 通用
generate_image.py 底层模型许可证未声明，版权路线存在法律不确定性
失败类型
Gate 0 准入缺陷 × 版权合规
症状
SKILL.md 仅说明调用 uv run scripts/generate_image.py，未说明底层生图模型名称、版本、训练数据授权、生成图像商用许可证
修复指令
在 SKILL.md 新增「版权声明」章节，填写：底层模型名称/版本；训练数据授权；生成图像商用状态；README 中"无版权问题"需补充法律依据
防重现
准入 Checklist 新增：使用 AI 生图功能的 Skill 必须在 SKILL.md 中声明模型名称及商用许可证，"内网 API"不等于"已授权商用"
KF-004 · Gate 0 · DIR-PPT · 通用
竞品对标完全缺失，幻灯片质量上限无外部锚点
失败类型
Gate 0 准入缺陷 × 竞品对标
症状
SKILL.md 无任何竞品对标声明，Gate 2 评测时无法判断"是否达到 Gamma/Tome 级别的幻灯片质量"
修复指令
在 SKILL.md 新增「竞品对标」章节：AI 幻灯片生成 → 对标 Gamma.app Pro；叙事型幻灯片 → 对标 Tome；智能布局 → 对标 Beautiful.ai；用相同 Mock Case 提示词测试竞品，截图存 reference/competitor-baseline/

DIR-PPT 准入项必填：已指定 ≥ 1 个外部竞品作为质量锚点，并提供同 Case 竞品截图
