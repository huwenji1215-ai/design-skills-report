Known Failures · 失败案例沉淀

本次评测发现的失败模式，按标准格式写入 Known Failures 库，供下一版本 Skill 准入规范更新使用。

KF-001 · AQE-A · DIR-01 · UI生成
字号体系约束缺失：Skill 未强制要求 ≥ 3级字号，模型自由裁量导致层级退化
失败类型
AQE-A（视觉层级不足）× DIR-01 UI生成 × 强制约束缺失
触发条件
Skill 未在 System Prompt 规范中明确「h1/h2/h3 三级字号」的强制要求，Claude 等模型倾向于用两级层次完成布局
症状
Claude 产物仅有 h1（主标题）和 h2（分类标题），工具卡片标题用 .card-title 类名但字号与分类标题差距不足2级，信息架构扁平，AQE-A 仅5.5
根因
SKILL.md 的视觉层级要求隐含在 product-thinking 中，未显式写出「字号体系 ≥ 3级（建议 32/22/16px）」的数值约束
修复指令
在 SKILL.md「生成规范」章节新增强制项：「字号层级 ≥ 3级，建议 h1:28-40px / h2:20-28px / h3:16-20px，各级差值 ≥ 4px」；并在 checklist.md P0 项中新增「是否存在至少3个可视区分的字号层级？」
防重现规则
准入 Checklist 新增 P0 项：「SKILL.md 是否明确写出字号层级数量要求（≥3级）及推荐数值？」
KF-002 · AQE-B · DIR-01 · UI生成
色彩规范缺位：无色相数量上限和情感调性引导，导致配色质量参差不齐
失败类型
AQE-B（色彩美学偏弱）× DIR-01 UI生成 × Skill 色彩规范缺位
触发条件
Skill 无「主色系 ≤ 5个色相」的硬性约束，也无「情感调性词（科技感/亲和力）」的场景引导，模型按自身默认偏好选色
症状
Claude 使用约6个色相且深色/浅色双模式情感调性不一致；Kimi/DeepSeek 虽色彩克制，但三模型配色雷同（均为GitHub深色），缺乏独特性。三模型 AQE-B 均值仅6.7，是最大失分项
根因
SKILL.md 无「色彩规范」独立章节；checklist.md 中色彩要求在「可选」项中，优先级不足；Skill 亦无竞品对标（无法从 v0.dev 等参照中学习色彩运用标准）
修复指令
在 SKILL.md 新增「色彩约束」章节：① 主色系 ≤ 5个色相，强调色 ≤ 2个；② 根据场景提供情感调性词（科技工具→深蓝/靛紫系；内容创作→暖橙/黄系）；③ 推荐 Tailwind / Open Color 调色板作为参考基准
防重现规则
准入 Checklist 新增 P1 项：「SKILL.md 是否限制了色相数量（≤5）？是否提供了场景化情感调性引导词？」

竞品对标完全缺失，质量锚点无从确立

Gate 0 准入缺陷 × 竞品对标缺失 × HTML原型类 Skill
