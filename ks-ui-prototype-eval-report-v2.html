Known Failures

KF-V2-001 · P1
Model B/C Toggle 开关无实际 JS 事件绑定
影响范围：系统设置页的通知开关 — 点击无法真实切换状态
根因：Prompt 要求"toggle 点击可切换"，但 B/C 输出仅用 CSS 实现外观，未绑定 click 事件
对比：Model A 正确实现（document.querySelectorAll('.toggle').forEach → addEventListener('click', ...)）
修复难度：低（3行 JS）
KF-V2-002 · P1
全部产物缺少 ARIA 属性
影响范围：图标按钮（通知bell）、导航项、状态 badge — 屏幕阅读器无法识别语义
根因：Prompt 中提及 WCAG AA 但未强制要求 aria-label；模型未主动添加
修复方案：为所有 icon-only 按钮添加 aria-label；导航 <nav> 添加 role="navigation"
修复难度：低（批量 grep + 添加属性）
KF-V2-003 · P2
折线图使用 div 宽度模拟，无真实数据点坐标
影响范围：数据概览页折线图 — 视觉有折线形态但数据点不可精确 hover
根因：Prompt 禁止 canvas，要求"CSS+JS 伪图表"，导致无法实现精确坐标交互
评估：属于 Prompt 约束导致的合理降级，不计入 AQE 扣分
KF-V2-004 · P2
Model A CSS 变量缺少 --font-size-* 和 --space-* 层级
影响范围：字号和间距直接写 px 值，维护时需全文搜索修改
对比：Model C 完整声明了 --space-1 到 --space-10，设计系统最规范
建议：Prompt 中明确要求"字号和间距必须用 CSS 变量"，增加约束强度

整改建议

🔧 FIX-01：Prompt 中强制 ARIA 属性（P1 修复）
在 Prompt 可访问性要求中补充："所有 icon-only 按钮必须添加 aria-label；导航区域必须使用 <nav role="navigation" aria-label="主导航">。" 通过 Prompt 约束层解决，无需 Skill 代码修改。
🔧 FIX-02：Toggle 事件绑定明确化（P1 修复）
Prompt 原文："toggle 开关可点击切换状态（JS）"——改为更明确的约束："toggle 开关必须用 JS 绑定 click 事件，切换 .on class，不得仅用 CSS :checked 或无事件绑定实现。" 可配合示例片段提升一致性。
🔧 FIX-03：CSS 变量完整性 Checklist 加入 QA Gate（P2 修复）
在 Skill 的 Gate 1 自检阶段加入 CSS 变量检查项：
--bg-base / --bg-surface / --bg-elevated ✓
--font-size-xs/sm/base/md/lg ✓
--space-1 ~ --space-10 ✓
如任一缺失，Skill 在输出前补全 CSS 变量声明块。
📈 FIX-04：升级建议 — 引入 Kimi/DeepSeek 端点测试多样性（架构改进）
本次评测中 Kimi endpoint (ep-3e24dk) 连续 connection reset，建议：① 检查 Kimi 端点的 max_tokens 上限；② 增加请求超时重试机制（3次，指数退避）；③ 在评测流水线中增加模型可用性健康检查。目标：确保 3 模型产物来自真正不同的模型架构，增加比较价值。
📈 FIX-05：下一版 Mock Case 方向建议
本次看板 Case 已覆盖多页面+图表+表格+表单。下一版建议：
方向A：移动端场景（手机壳 Viewer）— 如外卖 App 首页 + 购物车 + 个人中心
