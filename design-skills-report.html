它是做什么的

ui-ux-pro-max 是目前 skills.sh 上安装量最高的设计类技能（排除更通用类），定位为全栈 UI/UX 设计智能体。它的核心是一个可搜索的设计知识数据库，内置 50+ 设计风格（玻璃拟态、极简主义、暗黑模式、粗野主义等）、161 种调色板、57 种字体配对方案，以及 99 条有优先级的 UX 准则。通过 Python 搜索脚本（search.py），Agent 可以输入产品类型和关键词，精准检索匹配的设计系统推荐。

核心亮点
  • 可搜索数据库驱动：不只是规范文档，而是通过 `search.py --domain` 脚本实现按产品类型、风格、字体、颜色、UX准则分域检索，极大降低 Agent 查找设计规范的摩擦。
  • 设计系统持久化：支持 `--persist` 参数将设计决策写入 `design-system/MASTER.md`，跨会话保持一致性，解决 AI 设计前后风格不统一的痛点。
  • 10 大技术栈覆盖：React、Next.js、Vue、Svelte、SwiftUI、React Native、Flutter、Tailwind、shadcn/ui、HTML/CSS，一个技能覆盖 Web+Mobile 全场景。
  • 10 大规则类别带优先级：无障碍(CRITICAL)→触控(CRITICAL)→性能(HIGH)→风格→布局→排版→动画→表单→导航→图表，配合反模式列表，可直接用于 Code Review。
  • 交付前检查清单：4 大维度（视觉质量、交互、深浅色模式、无障碍）共 20+ 条 checkbox，可直接嵌入 CI 流程。

它是做什么的

这是 Vercel 官方出品的 Web 设计规范审计工具。它的工作模式完全不同于其他设计技能：它不是"生成设计"，而是"审查现有代码是否符合 Vercel Web Interface Guidelines"。每次调用时，它会先通过 WebFetch 实时拉取最新版规则（来自 GitHub raw），然后对指定文件进行逐条检查，输出 file:line 格式的精确问题报告。

核心亮点
  • 动态规则拉取：规则来源是 GitHub raw URL，每次执行都拉取最新版本，保证永不过时——这是目前同类技能中唯一实现"规范自更新"的。
  • 精准 file:line 输出：问题报告格式为 `components/Button.tsx:47 - 对比度不足，当前 3.2:1，需 ≥ 4.5:1`，直接可定位修复，无噪音。
  • Vercel 官方背书：规范来自 Vercel 内部 Web Interface Guidelines，这是 Next.js 生态最权威的 Web 设计标准之一。
  • 安装量最高（全设计类）：周装 181.7K，说明大量 Next.js/Vercel 生态开发者将其集成进了日常开发工作流。
  • Snyk 警告需注意：与其他技能不同，此技能 Snyk 审计为 Warn（其他两个官方技能为 Pass），需注意依赖链安全。

它是做什么的

Anthropic 官方出品，这是一份前端美学宣言，而非规范手册。它的核心使命是：消灭"AI Slop"美学（即每个 AI 生成的界面都长得一样的问题）。它要求 Agent 在编码前先回答四个设计问题（目的、风格基调、约束、差异化），然后选定一个大胆的美学方向，精准执行。GitHub Stars 高达 97.8K，是所有设计类技能中最高的，体现了社区对"反 AI 平庸设计"理念的强烈认同。

核心亮点
  • 差异性最强的设计理念：明确列出"禁止使用 Inter/Roboto/Arial"、"禁止白底紫渐变"、"禁止可预测的布局模式"——这些禁令直接针对 AI 生成设计的通病。
  • 设计思维框架：四问法（Purpose / Tone / Constraints / Differentiation），在编码之前强制建立美学共识，解决"先写代码再考虑设计"的坏习惯。
  • 极繁/极简双向兼容：明确说明"大胆的极繁主义"和"精致的极简主义"都是合法方向，关键是"意图性（intentionality）"而非强度，避免错误引导。
  • GitHub Stars 最高（97.8K）：说明它触达了大量开发者痛点，远超同类技能。
  • 三项安全审计全部通过：Gen Agent Trust Hub ✓、Socket ✓、Snyk ✓，是唯一全绿技能。

它是做什么的

canvas-design 是一款视觉艺术创作技能，Anthropic 官方出品。创作前必须先发明一套专属的"设计哲学运动"（如"Brutalist Joy"/"Chromatic Silence"），撰写 4-6 段设计宣言，然后以这套哲学为灵魂，创作出 90% 视觉、10% 文字的博物馆级单页或多页 PDF/PNG 艺术作品。

核心亮点
  • 设计哲学先行：每件作品都有专属宣言，彻底避免 AI 批量复制感，差异性评分最高（9.5）。
  • 博物馆/杂志级标准：明确要求"看起来像专业大师耗费数小时打磨"的输出标准。
  • 隐晦概念嵌入：将请求中的隐晦概念无形地嵌入抽象构图——"了解者能感受到，不了解者也欣赏其美感"。
  • 多页作品支持：支持咖啡桌书风格的多页 PDF，每页是同一哲学的不同变奏。

它是做什么的

专门针对仪表盘、管理后台和工具类 App 的设计技能（不适用于落地页/营销站点）。通过领域探索（找 5+ 概念词汇、5+ 领域色彩、1 个签名元素）和 Token 架构（5 类语义色彩变量），让每个界面都从产品世界中独立涌现，而非套模板。

核心亮点
  • 产品领域探索机制：设计前强制进行领域调研，提取概念词汇、色彩和签名元素——反模板化的核心武器。
  • Token 命名哲学：变量名反映产品世界（如 --ink / --parchment 而非 --gray-700），设计语言融入代码命名。
  • 挤眼测试 / 签名测试：独特的设计验证方法——模糊双眼后仍能感知层次，说明深度真实可信。
  • 命令式工作流：提供 :status、:audit、:extract、:critique 命令，结构化设计审查。

它是做什么的

一个外部 API 调用封装，通过调用 sleek.design REST API，用自然语言描述移动端设计需求，自动渲染并返回 PNG/WebP 截图。流程：创建项目 → 发送自然语言 → 轮询状态 → 获取截图。

核心亮点
  • 完全自然语言驱动：从"设计一个健身应用"到"在这个界面添加定价区域"，无需任何设计工具知识。
  • 真实截图交付：输出实际渲染的移动端截图（PNG/WebP、透明背景），可直接用于演示或 PRD。
  • 异步轮询架构：支持异步/同步两种模式，适应不同场景需求。

评测总结
洞察与行业观察

从 skills.sh 全量扫描来看，设计类技能正在从"规范文档"走向"智能化设计引擎"。本次评测的 6 款技能呈现出三种截然不同的演进方向：

方向一：设计知识数据库化（ui-ux-pro-max 代表）
将设计规范转化为可编程、可搜索的数据库，Agent 可以通过脚本精准检索匹配的设计方案。这是目前最实用的演进方向，降低了 AI 使用设计知识的摩擦。
方向二：美学理念化（frontend-design / interface-design 代表）
不只给出"怎么做对"的规范，而是建立"怎么做出独特感"的理念框架。这类技能意识到 AI 生成设计的最大问题不是不合规，而是千篇一律——它们通过强制建立美学意图来解决这个问题。
方向三：设计即代码审查（web-design-guidelines 代表）
将设计规范直接与代码 CI/CD 流程结合，以 file:line 格式输出问题，让设计合规变成可自动化的代码审查任务。这是工程化程度最高、最适合团队协作的方向。
核心洞察

最值得关注的现象是：安装量最高的两款技能（web-design-guidelines 181.7K + frontend-design 177.4K）都来自官方机构（Vercel Labs / Anthropic），说明开发者在设计类技能上有强烈的权威性偏好。而 ui-ux-pro-max 的 46.1K Stars 和 71.1K 周装量，则证明了"可搜索数据库驱动"这一设计理念正在获得广泛认可。

另外，interface-design 虽然 Stars 较少（4.1K），但其"领域探索→Token 架构→签名测试"的方法论深度在同类中独树一帜，是目前在 B2B 产品设计领域最有价值的非官方技能，值得重点关注。
